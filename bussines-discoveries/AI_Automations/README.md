# Deepwork AI Flows - Projeto de Negócio

## 📋 Visão Geral

Este repositório contém a documentação e planejamento para o negócio de **Deepwork AI Flows** (anteriormente AI Automations), uma parceria entre **Rodrigo Kugler** (lado comercial) e **João Balzer** (lado técnico - Engenheiro de Dados Sênior).

### Objetivo
**Empresa de ganho de performance via integrações e automações**

Criar **agentes/funcionários digitais** que usam as ferramentas que o cliente já tem (WhatsApp, e-mail, Slack, sistemas existentes) para automatizar processos específicos. **Não vendemos software novo, vendemos resultado**: menos trabalho manual, menos erro, menos folha, mais velocidade e organização.

### Modelo de Negócio
- **Produto**: Agente/Funcionário Digital (automação inteligente)
- **Posicionamento**: Empresa de performance, não de software tradicional
- **Abordagem**: Consultoria + automação (entendem processos, mapeiam, constroem agente)
- **Modelo de Receita**: Subscription (mensalidade recorrente)
- **Estratégia**: Nichar forte - escolher segmento específico e atacar dor MUITO específica

   - Usar `07_Operacional/ROTEIRO_ENTREVISTA.md` como guia

3. **Fazer Primeira Reunião com Cliente**
   - Explicar: "Funcionário digital que usa ferramentas que vocês já têm"

   - Preencher `07_Operacional/TEMPLATE_RELATORIO_CLIENTE.md`

     - 1-3 processos candidatos a MVP (bem descritos)
     - Lista de sistemas/telas/arquivos envolvidos
     - Estimativa de tempo/hora humana gasta
     - Custo mensal (salário, impostos)

### FASE 2: Desenvolvimento Técnico (JOÃO BALZER)

5. **Organizar Repositório GitHub** ✅
   - Estrutura: `/docs`, `/agents`, `/integrations`
   - GitHub como hub de tudo (código, docs, materiais)

6. **Desenhar Arquitetura Padrão de Agente**
   - Fluxo: Entrada → Interpretação/IA → Ações → Log → Supervisão
   - Definir stack mínima (nuvem, orquestrador, autenticação)

7. **Pegar Case da Concessionária como Exercício**
   - Pensar: Como automatizar rotina da contadora?
   - Que integrações precisaria?
   - Qual MVP mínimo que gera valor real?

8. **Estimar Esforço/Complexidade**
   - Quando Rodrigo trouxer material do cliente:
     - MVP (o que dá pra entregar primeiro)
     - V2 (melhorias e extras)
     - Prazo e "peso" de cada automação

### FASE 3: Proposta de Valor e Vendas (RODRIGO)

9. **Esboçar Propostas de Valor**
   - "Se automatizar X, você economiza ~R$Y/mês"
   - Hipótese de preço: % da economia ou valor fixo que "faça vergonha" perto do custo atual

10. **Construir Discurso de Venda Padrão**
    - Narrativa: "Empresa de performance", "funcionário digital"
    - "Usa o que você já tem", "consultoria + automação"
    - "Mensalidade pela manutenção do agente"

11. **Manter Relacionamento com Cliente**
    - Deixar claro que é MVP
    - Explicar que querem testar e mostrar valor
    - Depois transformar em contrato recorrente
